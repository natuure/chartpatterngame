# 프로젝트 현황 (핸드오프 노트)

Claude Code로 작업을 이어가기 위한 요약입니다. 이전 대화(Cowork)에서 결정된 내용과 남은 작업을 정리했습니다.

## 한 줄 컨셉

일정 기간의 실제 차트(캔들+거래량)를 보여주고 "다음 캔들이 위로 갈지 아래로 갈지" 맞히는 캐주얼 모바일 게임. 친구들끼리 모여서 같이 풀고 즐기는 용도. 종목명/날짜는 가려서 패턴 자체로만 승부하게 함.

## 핵심 게임 메커닉 (확정)

- 타임프레임: 일봉(1d) / 주봉(1w) / 3분봉(3m) 선택 가능
- 문제 출제 기준: 직전 종가 → 이벤트 캔들 종가 변동률(갭 포함)이 일봉 ±10%, 주봉 ±30%, 3분봉 ±5% 이상인 캔들만 채택. 상승/하락 동일 비율로 섞어서 정답이 한쪽으로 쏠리지 않게 함
- 이벤트 캔들 직전 5개 캔들(+거래량)을 먼저 보여줌 → 사용자가 위/아래 선택 → '플레이' 버튼 → 0.7초당 캔들 1개씩 재생(5개 = 3.5초) → 이벤트 캔들이 드라마틱하게 등장하며 정답 공개
- 한 세트 = 5/10/15라운드 선택, 정답 수·연속 정답(streak) 기록, 결과 화면에서 공유
- 종목명/날짜는 UI에 노출하지 않음 (패턴 학습 목적 + 투자자문 이슈 회피)

## 데이터 소스 (가장 중요한 미해결 과제)

국내(KR) + 미국(US) 주식 대상. 코인은 채택하지 않음(사용자 요청).

| 시장 | 타임프레임 | 소스 | 상태 |
|---|---|---|---|
| 국내 | 일봉/주봉 | pykrx (무료, 키 불필요) | 스크립트 작성됨, **사용자 PC에서 직접 실행 필요** (Claude 작업 환경은 KRX 접속 차단됨) |
| 국내 | 3분봉 | 한국투자증권 KIS Developers API | 스크립트 작성됨(검증 안 됨), **APP KEY/SECRET 발급 필요** |
| 미국 | 일봉/주봉/3분봉(1분봉 리샘플) | Twelve Data API | 스크립트 작성됨, **무료 API 키 발급 필요** (twelvedata.com) — 무료 플랜은 미국 주식만 지원, 분당 8회/일 800회 제한 |

API 키 발급 후 `scripts/build_question_bank.py` 를 돌리면 `app/questions-data.js` 가 생성되고, 앱이 자동으로 샘플 데이터 대신 실데이터를 사용함.

## 파일 구조

```
차트패턴게임/
├── PROJECT_STATUS.md          (이 파일)
├── app/
│   ├── index.html             게임 본체 (단일 HTML, 모바일 친화적, 외부 의존성 없음)
│   ├── sample_questions.js    데모용 합성 데이터(36문제) — 실데이터 없을 때 자동 fallback
│   └── questions-data.js      (빌드 후 생성됨, 실데이터)
├── data/
│   └── SCHEMA.md              문제 1개의 JSON 스키마 정의
└── scripts/
    ├── kr_scan_pykrx.py       국내 일봉/주봉 스캐너
    ├── kr_scan_kis_minute.py  국내 3분봉 스캐너 (KIS API)
    ├── us_scan_twelvedata.py  미국 일봉/주봉/3분봉 스캐너
    └── build_question_bank.py 위 결과들을 app/questions-data.js로 병합
```

## 검증 상태

- `app/index.html`: jsdom으로 5라운드 풀 게임 플로우(화면 전환/선택/애니메이션/채점/결과)를 시뮬레이션해서 에러 없이 끝까지 도는 것 확인함
- `scripts/kr_scan_pykrx.py`, `us_scan_twelvedata.py`: 로직은 작성했지만 Claude 작업 환경의 네트워크 제한(야후파이낸스/KRX/Twelve Data 등 다수 금융 도메인 차단)으로 실제 데이터로 실행 검증을 못 함. 사용자 환경에서 첫 실행 시 결과를 확인해야 함
- `scripts/kr_scan_kis_minute.py`: 검증 전혀 안 됨, KIS API 응답 필드명이 문서와 다를 수 있어 한 종목으로 먼저 테스트 권장

## 배포 계획

- 1차: 웹앱(PWA)으로 링크 공유 → 반응 테스트
- 이후: Capacitor로 감싸서 iOS/Android 네이티브 빌드 → 스토어 제출
  - 구글 플레이: TWA/Capacitor 빌드 비교적 수월, 등록비 $25(1회)
  - 애플 앱스토어: "단순 웹뷰 래퍼" 거절 이슈 있을 수 있어 네이티브스러운 기능(오프라인 캐싱 등) 보강 필요, 연회비 $99

## 수익화 계획 (합의된 순서)

1. 광고(배너/전면) + 리워드 광고(추가 라운드, 라이프 등)로 가볍게 시작
2. 반응 좋으면 테마팩 DLC(국내/미국/코인 특별판, 시즌별 모음) 인앱결제 추가
3. 랭킹 시즌패스로 경쟁/리텐션 강화

검토했던 다른 옵션: 종목명 공개 유료화(거부감 우려로 "광고 보면 공개" 방식 추천), 코스메틱(테마/뱃지), 제휴 마케팅(증권사 계좌개설 연동), B2B 단체 라이센스. 실제 종목+날짜를 직접 추천하는 형태로 가면 자본시장법상 투자자문업 이슈가 있을 수 있어 출시 전 법률 자문 권장(과거 데이터 기반 사후 공개 게임 형태는 비교적 안전할 것으로 보임).
