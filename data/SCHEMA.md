# 문제 데이터 스키마

게임 한 라운드 = 문제(question) 1개. 모든 스캐너 스크립트(KR/US, 일봉/주봉/분봉)는 이 형식의 JSON 배열을 생성한다.

```json
{
  "id": "KR_1d_005930_20240715",
  "market": "KR",
  "timeframe": "1d",
  "direction": "up",
  "change_pct": 12.4,
  "lead_candles": [
    {"o": 71000, "h": 71500, "l": 70200, "c": 70800, "v": 1234000},
    {"o": 70800, "h": 71200, "l": 70500, "c": 71000, "v": 980000}
  ],
  "reveal_candles": [
    {"o": 72000, "h": 73200, "l": 71800, "c": 72900, "v": 1500000},
    {"o": 72900, "h": 75000, "l": 72700, "c": 74800, "v": 2400000},
    {"o": 74800, "h": 78000, "l": 74500, "c": 77600, "v": 3200000},
    {"o": 77600, "h": 80200, "l": 77000, "c": 79500, "v": 3800000},
    {"o": 79500, "h": 81500, "l": 78800, "c": 80900, "v": 5400000}
  ],
  "meta": {
    "symbol": "005930",
    "date": "2024-07-15",
    "source": "pykrx"
  }
}
```

## 필드 설명

- `market`: `"KR"` 또는 `"US"`
- `timeframe`: `"1d"`(일봉) / `"1w"`(주봉) / `"3m"`(3분봉)
- `direction`: reveal_candles 마지막 종가가 lead_candles 마지막 종가 대비 오르면 `"up"`, 내리면 `"down"` — 퀴즈 정답
- `change_pct`: lead 마지막 종가 → reveal 마지막 종가까지의 누적 변동률(%). 절댓값 기준 일봉 10%, 주봉 30%, 3분봉 5% 이상만 채택
- `lead_candles`: 선택 전에 미리 보여주는 캔들 30개 (시간 순, 위 예시는 2개만 표시)
- `reveal_candles`: '플레이' 클릭 후 0.7초 간격으로 하나씩 공개하는 캔들 5개. 이 구간의 누적 변동이 정답을 결정
- `meta`: 종목코드/날짜/소스 — UI에는 노출하지 않음(패턴만으로 맞히기 위함). 디버깅/중복 방지용

## 통합 파일

- `data/kr_questions.json` — 국내 일봉+주봉 (pykrx)
- `data/kr_questions_minute.json` — 국내 3분봉 (KIS Developers)
- `data/us_questions.json` — 미국 일봉+주봉+3분봉 (Twelve Data)

앱은 위 3개 파일을 합쳐서 하나의 문제 풀(pool)로 사용하고, 타임프레임 선택 시 해당 timeframe만 필터링한다.
